# 🧪 راهنمای تست پروژه کولیب

این فایل نحوه اجرای تست‌ها، ساختار آن‌ها و بخش‌هایی که توسط تست‌های خودکار بررسی می‌شوند را توضیح میدهد.


# ⚙️ اجرای تست‌ها

برای اجرای تمامی تست‌های پروژه، دستور زیر را در روت پروژه اجرا کنید:

```bash
python manage.py test
یا
python3 manage.py test
```


# 📋 بخش‌های تحت پوشش تست

در حال حاضر تست‌ های پروژه بخش‌ های زیر را پوشش میدهند:

## 🔐 Authentication و احراز هویت

بررسی عملکرد قابلیت‌ های احراز هویت شامل:

* ثبت‌ نام کاربران
* ورود کاربران
* اعتبارسنجی
* عملیات ها روی حساب کاربری


## Book مدل

بررسی صحت عملکرد مدل کتاب در دیتابیس، شامل:

* ایجاد رکورد
* ذخیره اطلاعات
* اعتبارسنجی مدل
* بررسی ارتباطات دیتابیس


## Question مدل

بررسی عملکرد مدل سوالات آموزشی، شامل:

* ایجاد سوال
* ذخیره اطلاعات
* اعتبارسنجی داده‌ها
* بررسی رفتار مدل در دیتابیس


## 🤖 مدل های چت هوش مصنوعی

تست مدل‌ های مربوط به گفتگوی هوش مصنوعی شامل:

* ایجاد گفتگو
* ایجاد پیام
* ارتباط بین Chat و Message
* ذخیره صحیح اطلاعات در دیتابیس


## 🧠 LLM API

بررسی ارتباط با مدل‌های زبانی بزرگ شامل:

* صحت فراخوانی API
* بررسی ساختار پاسخ
* اعتبارسنجی پاسخ دریافتی


# 📁 ساختار تست‌ها

هر Domain تست ‌های مستقل خود را دارد و از **Django TestCase** استفاده می‌کند.

در پروژه، بسته به حجم تست‌ ها، از یکی از ساختارهای زیر استفاده شده است:

```
app/
├── tests.py
```

یا

```
app/
└── tests/
    ├── test_chats.py
    ├── test_provider.py
    └── ...
```
